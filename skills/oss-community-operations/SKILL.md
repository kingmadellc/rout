# OSS Community Operations

Apply this skill when repository trust and contributor experience need improvement.

## Workflow

1. Audit community baseline.
- Inspect `CODE_OF_CONDUCT.md`, `CONTRIBUTING.md`, `SECURITY.md`, and issue/PR templates.
- Verify messaging matches current product identity.

2. Repair template quality.
- Ensure bug and feature templates request reproducible details.
- Add issue template config to guide users to correct channels.

3. Align governance files.
- Keep reporting channels, behavioral standards, and contributor instructions consistent.
- Avoid contradictory guidance between docs.

4. Verify discoverability.
- Link governance files from `README.md`.
- Ensure template references use existing files/commands.

## Quality Gates

- Files must be concise and actionable.
- Security reporting path must be private, not public issue only.
- Templates must include environment/context capture fields.
