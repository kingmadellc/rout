Architecture

How it works

Three moving parts: a watcher, a command registry, and handlers.

1

Agent loop

Claude reasons through your request with multi-turn tool use. Max 5 iterations per message, tool results capped at 2k chars. Every tool call is logged.

2

Safety gate

Three-tier classification: SAFE actions execute immediately, CONFIRM actions require yes/no via iMessage, DESTRUCTIVE actions are blocked. Pending confirmations expire after 1 hour.

3

Hybrid transport

BlueBubbles push via Socket.IO with proof-of-life guard. Polling runs alongside push until delivery is proven, auto-falls back if push goes silent.

4

Crash-proof service

Runs as a macOS launchd service. Auto-restarts after crashes. PID lockfiles prevent duplicate processes. Texts you if it goes down.
